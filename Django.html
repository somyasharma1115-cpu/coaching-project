🎯 Complete Django – Build Web Apps Quickly

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. This course covers everything: setup, models, views, templates, URLs, forms, authentication, admin, and deployment. Let's build real web apps!

🚀 1. Django Fundamentals & Setup

🔹 What is Django?

MVT pattern (Model-View-Template). Batteries-included: ORM, auth, admin, forms, security. Used by Instagram, Pinterest, Disqus.

🔹 Installation & Project Setup

pip install django
django-admin startproject myproject
cd myproject
python manage.py runserver
Creates project structure. Settings, urls, wsgi. Run server at localhost:8000.

🔹 Creating an App

python manage.py startapp blog
# Add 'blog' to INSTALLED_APPS in settings.py
Apps are reusable modules. Each app has models, views, templates.

🗄️ 2. Models & Database (ORM)

🔹 Defining Models

from django.db import models

class Post(models.Model):
title = models.CharField(max_length=200)
content = models.TextField()
created_at = models.DateTimeField(auto_now_add=True)
def __str__(self): return self.title
Each model is a Python class that maps to a database table.

🔹 Migrations

python manage.py makemigrations
python manage.py migrate
makemigrations creates migration files. migrate applies them to DB.

🔹 Querying Data (ORM)

Post.objects.all()
Post.objects.filter(title__icontains="hello")
post = Post.objects.get(id=1)
post.save()
post.delete()
CRUD operations. Filters, get, create, update, delete.

🔹 Relationships

class Comment(models.Model):
post = models.ForeignKey(Post, on_delete=models.CASCADE)
author = models.CharField(max_length=100)

class Profile(models.Model):
user = models.OneToOneField(User, on_delete=models.CASCADE)

class Tag(models.Model):
posts = models.ManyToManyField(Post)
ForeignKey (many-to-one), OneToOneField, ManyToManyField.

🌐 3. Views & URL Routing

🔹 Function-Based Views (FBV)

from django.shortcuts import render
from .models import Post

def post_list(request):
posts = Post.objects.all()
return render(request, 'blog/post_list.html', {'posts': posts})
View takes request, returns HttpResponse or render template.

🔹 Class-Based Views (CBV)

from django.views.generic import ListView, DetailView

class PostListView(ListView):
model = Post
template_name = 'blog/post_list.html'
context_object_name = 'posts'
Powerful built-in generic views: ListView, DetailView, CreateView, UpdateView, DeleteView.

🔹 URL Configuration

# urls.py in app
from django.urls import path
from . import views

urlpatterns = [
path('', views.post_list, name='post_list'),
path('post/<int:pk>/', views.post_detail, name='post_detail'),
]
Map URLs to view functions. Named URLs for reverse().

🔹 Including Project URLs

# project/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
path('admin/', admin.site.urls),
path('blog/', include('blog.urls')),
]
Include app URLs. Namespacing with app_name.

🎨 4. Templates & Static Files

🔹 Template Language

<h1>{{ post.title }}</h1>
<p>{{ post.content|truncatewords:30 }}</p>
{% if user.is_authenticated %}
Hello {{ user.username }}
{% endif %}
{% for post in posts %}
<li>{{ post.title }}</li>
{% endfor %}
Variables {{ }}, tags {% %}, filters {{ value|filter }}. Inheritance with {% extends %} and {% block %}.

🔹 Template Inheritance

<!-- base.html -->
<!DOCTYPE html>
<html>
<head><title>{% block title %}My Site{% endblock %}</title></head>
<body>{% block content %}{% endblock %}</body>
</html>

<!-- child.html -->
{% extends "base.html" %}
{% block content %}...{% endblock %}
DRY templates. Blocks can be overridden.

🔹 Static Files (CSS, JS, Images)

# settings.py
STATIC_URL = '/static/'
STATICFILES_DIRS = [BASE_DIR / "static"]

# template
{% load static %}
<link rel="stylesheet" href="{% static 'css/style.css' %}">
Collect static files for production. Media for user uploads.

📝 5. Forms & ModelForms

🔹 Creating Forms

from django import forms

class ContactForm(forms.Form):
name = forms.CharField(max_length=100)
email = forms.EmailField()
message = forms.CharField(widget=forms.Textarea)
HTML form rendering, validation, CSRF protection.

🔹 ModelForm

from django.forms import ModelForm
from .models import Post

class PostForm(ModelForm):
class Meta:
model = Post
fields = ['title', 'content']
Automatically generate form from model. Saves time.

🔹 Handling Form in View

def create_post(request):
if request.method == 'POST':
form = PostForm(request.POST)
if form.is_valid():
form.save()
return redirect('post_list')
else:
form = PostForm()
return render(request, 'blog/post_form.html', {'form': form})
GET shows empty form. POST validates and saves.

🔐 6. Authentication System

🔹 Built-in User Model

from django.contrib.auth.models import User
user = User.objects.create_user(username='john', password='secret')
User model with username, password, email, etc.

🔹 Login, Logout, Registration

from django.contrib.auth import login, logout, authenticate
from django.contrib.auth.forms import UserCreationForm

# Login view
user = authenticate(request, username=username, password=password)
if user: login(request, user)
# Logout
logout(request)
Use built-in auth views or create custom.

🔹 Protecting Views

from django.contrib.auth.decorators import login_required

@login_required
def profile(request):
return render(request, 'profile.html')
Require login. Also use LoginRequiredMixin for CBV.

🔹 User Groups & Permissions

user.groups.add(group)
user.has_perm('app.can_publish')
@permission_required('app.can_publish')
Fine-grained access control.

👑 7. Django Admin

🔹 Registering Models

# admin.py
from django.contrib import admin
from .models import Post

admin.site.register(Post)
Superuser: python manage.py createsuperuser. Access /admin.

🔹 Customizing Admin

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
list_display = ('title', 'created_at', 'author')
list_filter = ('created_at',)
search_fields = ('title', 'content')
prepopulated_fields = {'slug': ('title',)}
Custom list view, filters, search, inline editing.

⚡ 8. Advanced Django Topics

🔹 Class-Based Views (Detail, Create, Update, Delete)

from django.views.generic.edit import CreateView, UpdateView, DeleteView
from django.urls import reverse_lazy

class PostCreateView(CreateView):
model = Post
fields = ['title', 'content']
success_url = reverse_lazy('post_list')
Generic editing views with automatic form handling.

🔹 Middleware

class SimpleMiddleware:
def __init__(self, get_response):
self.get_response = get_response
def __call__(self, request):
# request preprocessing
response = self.get_response(request)
# response postprocessing
return response
Intercept request/response. Built-in: authentication, security, session.

🔹 Signals

from django.db.models.signals import post_save
from django.dispatch import receiver

@receiver(post_save, sender=User)
def create_profile(sender, instance, created, **kwargs):
if created:
Profile.objects.create(user=instance)
Allow decoupled applications to get notified when actions occur.

🔹 Pagination

from django.core.paginator import Paginator

posts = Post.objects.all()
paginator = Paginator(posts, 10)
page_number = request.GET.get('page')
page_obj = paginator.get_page(page_number)
Split large querysets into pages.

🚀 9. Deployment & Production

🔹 Settings for Production

DEBUG = False
ALLOWED_HOSTS = ['yourdomain.com']
DATABASES = { 'default': { 'ENGINE': 'django.db.backends.postgresql', ... } }
STATIC_ROOT = BASE_DIR / 'staticfiles'
Disable debug, set allowed hosts, use PostgreSQL, collectstatic.

🔹 Environment Variables

import os
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY')
# Use python-decouple or django-environ
Never hardcode secrets. Use env variables.

🔹 Deploy with Gunicorn & Nginx

pip install gunicorn
gunicorn myproject.wsgi:application

# Nginx reverse proxy config
# serve static files via Nginx
WSGI server + web server. Use whitenoise for static files.

🔹 Debugging & Logging

import logging
logger = logging.getLogger(__name__)
logger.error("Something went wrong")
Log errors, use Django Debug Toolbar for development.

📡 10. Django REST Framework (DRF)

🔹 Installation & Serializers

pip install djangorestframework

from rest_framework import serializers
class PostSerializer(serializers.ModelSerializer):
class Meta:
model = Post
fields = ['id', 'title', 'content']
Convert models to JSON/XML for APIs.

🔹 API Views & Routers

from rest_framework.viewsets import ModelViewSet

class PostViewSet(ModelViewSet):
queryset = Post.objects.all()
serializer_class = PostSerializer

# router registers URLs automatically
CRUD API endpoints with minimal code. Browsable API.
