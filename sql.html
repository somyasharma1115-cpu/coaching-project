🗄️ Complete SQL – Manage & Query Data Like a Pro

SQL (Structured Query Language) is the standard language for relational databases. This course covers everything: SELECT, filtering, sorting, joins, aggregations, subqueries, indexes, transactions, stored procedures, and more. Clear examples for MySQL/PostgreSQL. Let's master data!

📖 1. SQL Fundamentals

🔹 What is SQL?

SQL is used to communicate with databases. Popular DBMS: MySQL, PostgreSQL, SQLite, SQL Server, Oracle.

🔹 Database, Table, Record

Database contains tables. Tables have rows (records) and columns (fields). Primary key uniquely identifies each row.

🔹 Creating a Database & Table

CREATE DATABASE school;
USE school;

CREATE TABLE students (
id INT PRIMARY KEY AUTO_INCREMENT,
name VARCHAR(100) NOT NULL,
age INT,
grade VARCHAR(2)
);

✍️ 2. CRUD Operations

🔹 INSERT – Add Data

INSERT INTO students (name, age, grade)
VALUES ('Alice', 20, 'A'), ('Bob', 22, 'B');

🔹 SELECT – Read Data

SELECT * FROM students;
SELECT name, age FROM students WHERE age > 20;

🔹 UPDATE – Modify Data

UPDATE students SET grade = 'A+' WHERE name = 'Alice';

🔹 DELETE – Remove Data

DELETE FROM students WHERE id = 2;
-- DELETE without WHERE removes all rows

🔍 3. Filtering & Sorting

🔹 WHERE Clause

SELECT * FROM products
WHERE price > 100 AND category = 'Electronics';

Operators: =, <>, >, <, >=, <=, BETWEEN, LIKE, IN

🔹 LIKE & Wildcards

SELECT * FROM users WHERE name LIKE 'J%'; -- starts with J
SELECT * FROM users WHERE email LIKE '%@gmail.com';

🔹 ORDER BY

SELECT * FROM products ORDER BY price DESC, name ASC;

🔹 LIMIT & OFFSET

SELECT * FROM orders LIMIT 10 OFFSET 20; -- pagination

📊 4. Aggregate Functions & Grouping

🔹 Aggregate Functions

SELECT COUNT(*) FROM students;
SELECT AVG(price), MAX(price), MIN(price), SUM(quantity) FROM sales;

🔹 GROUP BY

SELECT category, COUNT(*) as product_count
FROM products
GROUP BY category;

🔹 HAVING (filter groups)

SELECT category, AVG(price) as avg_price
FROM products
GROUP BY category
HAVING avg_price > 50;

🔗 5. Joins – Combining Tables

🔹 INNER JOIN

SELECT orders.id, customers.name
FROM orders
INNER JOIN customers ON orders.customer_id = customers.id;
Returns rows with match in both tables.

🔹 LEFT JOIN (RIGHT JOIN)

SELECT customers.name, orders.total
FROM customers
LEFT JOIN orders ON customers.id = orders.customer_id;
All rows from left table, matching from right (NULL if none).

🔹 FULL OUTER JOIN

-- MySQL doesn't support directly; use UNION of LEFT and RIGHT JOIN

🔹 SELF JOIN

SELECT e1.name AS employee, e2.name AS manager
FROM employees e1
LEFT JOIN employees e2 ON e1.manager_id = e2.id;

📦 6. Subqueries & Nested Queries

🔹 Subquery in WHERE

SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

🔹 Subquery with IN

SELECT name FROM customers
WHERE id IN (SELECT customer_id FROM orders WHERE total > 100);

🔹 EXISTS / NOT EXISTS

SELECT name FROM products p
WHERE EXISTS (SELECT 1 FROM inventory i WHERE i.product_id = p.id);

🔹 Subquery in SELECT (scalar)

SELECT name,
(SELECT AVG(salary) FROM employees) as company_avg
FROM employees;

🔄 7. Set Operations

🔹 UNION & UNION ALL

SELECT name FROM employees
UNION
SELECT name FROM contractors;
-- UNION removes duplicates, UNION ALL keeps all

🔹 INTERSECT & EXCEPT (MINUS)

-- INTERSECT: common rows
-- EXCEPT: rows in first not in second

⚡ 8. Indexes & Query Optimization

🔹 Creating Indexes

CREATE INDEX idx_name ON students (name);
CREATE UNIQUE INDEX idx_email ON users (email);
DROP INDEX idx_name ON students;
Indexes speed up SELECT but slow INSERT/UPDATE/DELETE.

🔹 EXPLAIN Query Plan

EXPLAIN SELECT * FROM orders WHERE customer_id = 5;

🔒 9. Constraints & Keys

🔹 PRIMARY KEY & FOREIGN KEY

CREATE TABLE orders (
id INT PRIMARY KEY,
customer_id INT,
FOREIGN KEY (customer_id) REFERENCES customers(id)
);

🔹 UNIQUE, NOT NULL, CHECK

CREATE TABLE users (
username VARCHAR(50) UNIQUE,
email VARCHAR(100) NOT NULL,
age INT CHECK (age >= 18)
);

🚀 10. Advanced Topics

🔹 Views (Virtual Tables)

CREATE VIEW active_users AS
SELECT id, name FROM users WHERE status = 'active';
-- Use like a table: SELECT * FROM active_users;

🔹 Stored Procedures

DELIMITER //
CREATE PROCEDURE GetUsersByAge(IN min_age INT)
BEGIN
SELECT * FROM users WHERE age >= min_age;
END //
CALL GetUsersByAge(18);

🔹 Transactions (ACID)

START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT; -- or ROLLBACK if error
Ensures data integrity. COMMIT saves changes, ROLLBACK undoes.

🔹 Window Functions (ROW_NUMBER, RANK)

SELECT name, salary,
ROW_NUMBER() OVER (ORDER BY salary DESC) as rank
FROM employees;

SELECT department, name, salary,
RANK() OVER (PARTITION BY department ORDER BY salary DESC) as dept_rank
FROM employees;

🔹 Common Table Expressions (CTE)

WITH high_salary AS (
SELECT * FROM employees WHERE salary > 50000
)
SELECT * FROM high_salary WHERE department = 'IT';

🔹 Triggers

CREATE TRIGGER update_timestamp
BEFORE UPDATE ON users
FOR EACH ROW
SET NEW.updated_at = NOW();

📚 SQL Cheat Sheet – Quick Reference

🔹 Query Order
SELECT → FROM → JOIN → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
🔹 Common Joins
INNER, LEFT, RIGHT, FULL, SELF
🔹 Aggregates
COUNT, SUM, AVG, MIN, MAX
🔹 Constraints
PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL, CHECK, DEFAULT
🔹 Transaction
START TRANSACTION, COMMIT, ROLLBACK, SAVEPOINT
🔹 Wildcards
% (any characters), _ (single character)
💡 Pro Tips: Always use indexes on columns used in WHERE, JOIN, ORDER BY. Prefer JOINs over subqueries when possible. Use EXPLAIN to analyze queries. Normalize data to reduce redundancy but denormalize for performance when needed.
