⏱️ 1. Complexity Analysis (Big O)

🔹 Time & Space Complexity

Big O notation describes how runtime grows with input size. Common complexities: O(1) constant, O(log n) logarithmic, O(n) linear, O(n log n), O(n²) quadratic, O(2^n) exponential.
# O(1) – constant time
def get_first(arr): return arr[0]

# O(n) – linear
def find_max(arr):
max_val = arr[0]
for x in arr: max_val = max(max_val, x)
return max_val

📊 2. Arrays & Strings

🔹 Array Operations

arr = [1,2,3,4,5]
arr.append(6) # O(1)
arr.pop() # O(1)
arr.insert(0, 99) # O(n)
arr.remove(3) # O(n)
print(arr[2]) # O(1) random access
Contiguous memory. Fast access (O(1)), slow insert/delete in middle (O(n)).

🔹 Two Pointers & Sliding Window

# Two pointers – reverse array
left, right = 0, len(arr)-1
while left < right:
arr[left], arr[right] = arr[right], arr[left]
left += 1; right -= 1
Techniques for subarray problems: sliding window for sums, two-pointer for pairs.

🔹 String Manipulation

s = "hello"
s.upper(), s.lower()
s.split()
''.join(reversed(s))
s[1:4] # slice
Strings are immutable in Python. Use list of chars for modifications.

🔗 3. Linked Lists

🔹 Singly Linked List

class Node:
def __init__(self, val):
self.val = val
self.next = None

# Insert at head: O(1)
# Insert at tail: O(n)
# Delete node: O(n) to find
Dynamic size, no contiguous memory. Fast insert/delete at ends, slow random access.

🔹 Doubly Linked List

class DNode:
def __init__(self, val):
self.val = val
self.prev = None
self.next = None
Each node has prev pointer. Easier to traverse backward.

🔹 Common Operations

# Reverse linked list
def reverse(head):
prev = None
curr = head
while curr:
nxt = curr.next
curr.next = prev
prev = curr
curr = nxt
return prev
Detect cycle (Floyd's cycle detection). Find middle node.

📦 4. Stacks & Queues

🔹 Stack (LIFO)

stack = [] # list as stack
stack.append(1) # push
top = stack.pop() # pop
peek = stack[-1] # O(1)
Used for undo/redo, expression evaluation, DFS, backtracking.

🔹 Queue (FIFO)

from collections import deque
q = deque()
q.append(1) # enqueue
first = q.popleft() # dequeue O(1)
Used in BFS, task scheduling, print spooler.

🔹 Monotonic Stack/Queue

Maintain increasing/decreasing order. Used for next greater element, sliding window max.

🔄 5. Recursion & Backtracking

🔹 Recursion Fundamentals

def factorial(n):
if n <= 1: return 1
return n * factorial(n-1)
Base case + recursive call. Stack usage, risk of stack overflow.

🔹 Backtracking

# Generate all subsets (backtracking)
def subsets(nums):
result = []
def backtrack(start, path):
result.append(path[:])
for i in range(start, len(nums)):
path.append(nums[i])
backtrack(i+1, path)
path.pop()
backtrack(0, [])
return result
Try all possibilities, undo (backtrack) when stuck. Used in N-Queens, permutations, subsets.

🔢 6. Sorting Algorithms

🔹 Bubble, Insertion, Selection

# Bubble sort O(n²)
for i in range(n):
for j in range(0, n-i-1):
if arr[j] > arr[j+1]:
arr[j], arr[j+1] = arr[j+1], arr[j]
Simple but inefficient for large data.

🔹 Merge Sort (Divide & Conquer)

def merge_sort(arr):
if len(arr) <= 1: return arr
mid = len(arr)//2
left = merge_sort(arr[:mid])
right = merge_sort(arr[mid:])
return merge(left, right)
O(n log n) time, O(n) space. Stable sort.

🔹 Quick Sort

def quick_sort(arr):
if len(arr) <= 1: return arr
pivot = arr[0]
left = [x for x in arr[1:] if x <= pivot]
right = [x for x in arr[1:] if x > pivot]
return quick_sort(left) + [pivot] + quick_sort(right)
Average O(n log n), worst O(n²) if bad pivot.

🔹 Heap Sort & Counting Sort

Heap sort: O(n log n), in-place. Counting sort: O(n+k) for small integer range.

🔍 7. Searching Algorithms

🔹 Linear Search

for i in range(len(arr)):
if arr[i] == target: return i
return -1 # O(n)

🔹 Binary Search (on sorted array)

def binary_search(arr, target):
left, right = 0, len(arr)-1
while left <= right:
mid = (left+right)//2
if arr[mid] == target: return mid
elif arr[mid] < target: left = mid+1
else: right = mid-1
return -1 # O(log n)
Divide and conquer. Requires sorted input.

🔹 Ternary Search & Exponential Search

Ternary search for unimodal functions. Exponential search for unbounded arrays.

🌳 8. Trees

🔹 Binary Tree & Traversals

class TreeNode:
def __init__(self, val):
self.val = val
self.left = None
self.right = None

# Inorder: left, root, right
# Preorder: root, left, right
# Postorder: left, right, root
DFS traversals: recursive or stack. BFS: level order using queue.

🔹 Binary Search Tree (BST)

def insert(root, val):
if not root: return TreeNode(val)
if val < root.val: root.left = insert(root.left, val)
else: root.right = insert(root.right, val)
return root
Left child < parent < right child. O(log n) average for search/insert.

🔹 Balanced Trees (AVL, Red-Black)

Self-balancing. AVL: strict height balance (log n). Red-Black: used in C++ map, Java TreeMap.

🔹 Heap (Priority Queue)

import heapq
heap = [3,1,4]
heapq.heapify(heap) # min-heap
heapq.heappush(heap, 2)
smallest = heapq.heappop(heap)
Complete binary tree. O(log n) push/pop. Used in Dijkstra, heap sort.

🔹 Trie (Prefix Tree)

Efficient for word storage & prefix search. Used in autocomplete, spell checker.

🌐 9. Graphs

🔹 Representations

# Adjacency list – most common
graph = {0: [1,2], 1: [0,3], 2: [0], 3: [1]}
# Adjacency matrix – dense graphs
Directed/Undirected, weighted/unweighted.

🔹 BFS & DFS

from collections import deque
def bfs(graph, start):
visited = set()
q = deque([start])
while q:
node = q.popleft()
if node not in visited:
visited.add(node)
q.extend(graph[node] - visited)
BFS: shortest path in unweighted. DFS: recursion or stack.

🔹 Shortest Paths

# Dijkstra (non-negative weights)
import heapq
def dijkstra(graph, start):
distances = {node: float('inf') for node in graph}
distances[start] = 0
pq = [(0, start)]
while pq:
dist, node = heapq.heappop(pq)
for neighbor, weight in graph[node]:
new_dist = dist + weight
if new_dist < distances[neighbor]:
distances[neighbor] = new_dist
heapq.heappush(pq, (new_dist, neighbor))
Bellman-Ford (negative weights). Floyd-Warshall (all-pairs).

🔹 Minimum Spanning Tree

Kruskal (union-find) & Prim's algorithm.

🔹 Topological Sort & Cycle Detection

Kahn's algorithm (BFS) or DFS. Used in task scheduling.

🧠 10. Dynamic Programming

🔹 Memoization (Top-Down)

def fib(n, memo={}):
if n in memo: return memo[n]
if n <= 1: return n
memo[n] = fib(n-1, memo) + fib(n-2, memo)
return memo[n]
Store results of subproblems to avoid recomputation.

🔹 Tabulation (Bottom-Up)

dp = [0]*(n+1)
dp[1] = 1
for i in range(2, n+1):
dp[i] = dp[i-1] + dp[i-2]
Iterative, often more efficient. Base cases to final result.

🔹 Classic DP Problems

Knapsack, Longest Common Subsequence, Edit Distance, Coin Change, Maximum Subarray (Kadane's), House Robber, Climbing Stairs, etc.

💎 11. Greedy Algorithms

🔹 Idea & Examples

Make locally optimal choice at each step. Works for problems like activity selection, fractional knapsack, Huffman coding, Dijkstra's (greedy in essence).
# Activity selection: choose earliest finishing times
activities.sort(key=lambda x: x[1]) # sort by end time
